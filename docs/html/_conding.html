convention

As you probably now javascript is not a strongly typed language and it's bit hard to get the type of an object while programing. To make your life easier we adopted a convention in the naming of the variables and functions.

Prefixes

  • e_: Enumeration 
     var e_state = states_e.START;
  • f_: Floating point number 
     f_density = 0.5;
  • i_: Integer number 
     var i_index = 1;
  • o_: Object created using new() 
     var o_stack = new tsip_stack_t(...);
  • ao_: Array of Objects 
     var ao_stacks = new Array();
  • on_: Callback function pointer 
     o_stack.on_event = function(e){};
  • s_: String 
     var s_uri = 'sip:alice@doubango.org';
  • as_: Array of strings 
     var ao_uris = new Array();
Please note that static variables are prefixes by "__" followed by the type e.g. @code var __i_timeout = 1800;

    Sufixes

  • _t: An object type which could be created using 'new' keyword.
  • _e: Enumeration type.
  • _f: Variable defining a function (like C/C++ function pointers or CSharp delegates)
